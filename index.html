1. Local Implementation Guide

Follow these steps to initialize the development environment on your local machine.

Backend Setup
cd backend
python -m venv venv
source venv/bin/activate # or venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload

Docs available at: http://127.0.0.1:8000/docs

Frontend Setup
cd onion-frontend
npm install
npm start

Accessible at: http://localhost:3000

2. Deep Learning Architecture

The core engine uses a Convolutional Neural Network (CNN) optimized for mobile deployment.

Layer Type Purpose Specification
Base Model Feature Extraction MobileNetV2 (Pre-trained)
Global Pooling Dimensionality Reduction GlobalAveragePooling2D
Regularization Prevent Overfitting Dropout (50%) & Batch Norm
Classifier Disease Prediction Dense (Softmax Activation)

The Training Pipeline

Phase 1: Transfer Learning

Base layers are frozen. Training focuses strictly on the new top-head layers for 10 epochs using the Adam optimizer to calibrate basic outputs.

Phase 2: Fine Tuning

Deeper layers of MobileNet are "thawed". Training continues for 5 epochs at a hyper-low learning rate (1e-5) to refine feature detection for onion-specific textures.

Note on Chat Functionality: The AI Assistant uses external API bounds connecting to google/flan-t5-base on Hugging Face; it is not hosted locally.

3. Production Deployment

Backend (API) Hosting

  1. Ensure onion_mobilenet_model.h5 and classes.json are tracked in Git.
  2. Connect repository to Render or Railway.
  3. Set Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT.

Frontend (UI) Hosting

  1. Update API calls in React to use your new live Backend URL via .env.
  2. Deploy to Vercel or Netlify.
  3. Critical Security: Update CORS settings in main.py:
    allow_origins = ["https://your-agrovision-app.vercel.app"]
